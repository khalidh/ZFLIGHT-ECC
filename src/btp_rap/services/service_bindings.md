# Service Bindings

Create these bindings in ADT after activating the service definitions.

| Service Definition | Binding Type | Suggested Binding | Usage |
|---|---|---|---|
| `ZF2_UI_FLIGHT_MANAGE` | OData V4 - UI | `ZF2_UI_FLIGHT_MANAGE_O4` | Fiori Elements carrier/flight app |
| `ZF2_UI_BOOKING_MANAGE` | OData V4 - UI | `ZF2_UI_BOOKING_MANAGE_O4` | Fiori Elements booking/order/invoice app |
| `ZF2_API_FLIGHT_BOOKING` | OData V4 - Web API | `ZF2_API_FLIGHT_BOOKING_O4` | External integration API |

After publishing, assign the UI services to an IAM app and business catalog.

## ADT Creation Steps

Create the bindings directly in Eclipse ADT:

1. Right-click package `ZFLIGHT-ECC3`.
2. Choose `New` > `Other ABAP Repository Object`.
3. Search `Service Binding`.
4. Create each binding listed below.
5. Activate the binding.
6. Open the binding and click `Publish`.

## Draft Activation Notes

The booking service is draft-enabled for the transactional entities:

- `ZF2_I_Booking` uses draft table `ZF2_D_BOOKING`.
- `ZF2_I_Order` uses draft table `ZF2_D_ORDER`.
- `ZF2_I_Invoice` uses draft table `ZF2_D_INVOICE`.
- `ZF2_I_Payment` uses draft table `ZF2_D_PAYMENT`.

On the first activation, ADT can report that a draft table does not exist. Use the ADT Quick Fix on the behavior definition to generate the missing draft table, then activate in this order:

1. Active tables, including `ZF2_PAYMENT` after adding `LAST_CHANGED_AT`.
2. Interface CDS.
3. Projection CDS.
4. Interface behavior definitions.
5. Projection behavior definitions.
6. Service definitions.
7. Service bindings, then publish again.

After publishing `ZF2_UI_BOOKING_MANAGE_O4`, its `$metadata` should contain `DraftRoot` for `Bookings`, `Orders`, `Invoices`, and `Payments`.

### Draft Metadata Checklist For Missing Create/Edit Buttons

If the Fiori Elements preview shows `Delete` but does not show `Create` or `Edit`, do not diagnose the UI annotations first. Check the draft metadata generated by the binding.

| Order | Object or place | Check | Expected result | If not OK |
|---:|---|---|---|---|
| 1 | `ZF2_I_BOOKING` BDEF | `with draft;` | Interface behavior is draft-enabled | Add draft support and activate |
| 2 | `ZF2_I_BOOKING` BDEF | `draft table zf2_d_booking` | Draft table is declared | Generate the draft table with ADT Quick Fix |
| 3 | `ZF2_D_BOOKING` table | Object exists and is active | Draft persistence is available | Generate and activate it |
| 4 | `ZF2_I_BOOKING` BDEF | `create;`, `update;`, `delete;` | RAP CRUD operations are declared | Correct and activate the BDEF |
| 5 | `ZF2_I_BOOKING` BDEF | `draft action Edit`, `Activate`, `Discard`, `Resume`, `Prepare` | Draft lifecycle actions are declared | Correct and activate the BDEF |
| 6 | `ZF2_C_BOOKING` BDEF | `projection;` and `use draft;` | Projection is draft-enabled | Correct and activate the projection BDEF |
| 7 | `ZF2_C_BOOKING` BDEF | `use create;`, `use update;`, `use delete;` | CRUD is exposed to the UI service | Correct and activate the projection BDEF |
| 8 | `ZF2_C_BOOKING` CDS | `define root view entity` and `provider contract transactional_query` | Projection is transactional | Correct and activate the CDS |
| 9 | `ZF2_UI_BOOKING_MANAGE` SRVD | `expose ZF2_C_Booking as Bookings;` | `Bookings` is exposed by the service | Correct and activate the SRVD |
| 10 | `ZF2_UI_BOOKING_MANAGE_O4` SRVB | Binding is active and published | Current service metadata is published | Publish or republish the binding |
| 11 | `$metadata` | `Common.DraftRoot` on `Bookings` | Fiori recognizes `Bookings` as a draft root | Reactivate BDEF/CDS/SRVD, then republish |
| 12 | `$metadata` | `NewAction` in `DraftRoot` | Fiori can render `Create` | Check create/use create, draft table and republish |
| 13 | `$metadata` | `EditAction` in `DraftRoot` | Fiori can render `Edit` on the object page | Check update/use update and draft `Edit` action |
| 14 | `$metadata` | `ActivationAction`, `DiscardAction`, `PreparationAction` | Draft save/discard/prepare flow is complete | Check draft actions and republish |
| 15 | `$metadata` | `InsertRestrictions`, `UpdateRestrictions`, `DeleteRestrictions` | Capabilities do not block the expected operation | Remove or correct restrictive annotations |

In a draft-enabled Fiori Elements V4 app, `Edit` usually appears on the object page after opening a row, not necessarily in the list report toolbar. `Create` depends especially on `Common.DraftRoot/NewAction`.

## Validation Checklist

- All three service bindings are active.
- All three service bindings are published.
- Preview works for `ZF2_UI_FLIGHT_MANAGE_O4`.
- Preview works for `ZF2_UI_BOOKING_MANAGE_O4`.
- `$metadata` for `ZF2_UI_BOOKING_MANAGE_O4` contains `Common.DraftRoot/NewAction` for `Bookings`.
- `$metadata` for `ZF2_UI_BOOKING_MANAGE_O4` contains `Common.DraftRoot/EditAction` for `Bookings`.
- The metadata document of `ZF2_API_FLIGHT_BOOKING_O4` is reachable.
- Actions appear in the service metadata for booking, order and invoice.

## If Creation Is Blocked

- Re-activate the service definition first.
- Re-activate the projection CDS used by the service.
- Re-activate the behavior projection for the same projection CDS.
- Re-open the service binding editor after activation.
